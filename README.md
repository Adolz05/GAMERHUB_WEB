## 📝 Descripción del Proyecto

**GamerHub** nace para solucionar un problema muy común en el entorno gaming actual: la falta de espacios centralizados y organizados para encontrar compañeros de juego con los mismos intereses, horarios disponibles y niveles de experiencia. 

Esta plataforma web actúa como el escaparate principal de la aplicación, permitiendo a los usuarios visualizar el funcionamiento del sistema, los roles de acceso, la gestión de datos mediante formularios y la generación de informes interactivos.

---

## ✨ Características Clave

* **👥 Gestión de Comunidades:** Espacios dedicados por cada videojuego (públicos o privados).
* **⚡ Misiones y Desafíos:** Planificación de eventos colaborativos con control de estado (pendiente/completado).
* **🏆 Rankings e Informes:** Consulta de líderes, históricos de eventos y estadísticas en tiempo real.
* **🛡️ Sistema de Permisos:** Control de accesos estructurado en 3 roles (Usuario Estándar, Moderador y Administrador).

---

## 🛠️ Tecnologías Utilizadas

Para lograr un diseño responsivo, fluido y con una estética puramente *gaming*, se han empleado las siguientes tecnologías:

* **HTML5 & CSS3:** Estructura semántica y personalización de variables de color (`--accent`, `--primary`).
* **Bootstrap v5.3:** Maquetación rápida, sistema de rejilla (*Grid*) y componentes responsivos.
* **Bootstrap Icons:** Biblioteca de iconos vectoriales para la interfaz.
* **AOS (Animate On Scroll):** Biblioteca de JavaScript para las transiciones y animaciones visuales al hacer scroll.
* **JavaScript (Vanilla):** Lógica integrada para el menú móvil, control de scroll-top y tracking activo de navegación.

---

## 🗄️ Arquitectura de Datos (Entidades)

El sistema está preparado para escalar con un backend relacional estructurado en **5 entidades principales**:

1.  **Usuarios:** Gestión de perfiles, rangos, credenciales y roles.
2.  **Videojuegos:** Catálogo centralizado con filtros por género y plataformas (PC, PS5, Xbox...).
3.  **Comunidades:** Grupos de jugadores asociados a un título específico y liderados por un administrador.
4.  **Misiones / Desafíos:** Eventos fechados con niveles de dificultad y control de usuarios inscritos.
5.  **Roles y Permisos:** Matriz de control de accesos para la seguridad de la plataforma.
